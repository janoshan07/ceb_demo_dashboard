Single-module Spring Boot application (`BillingApplication` as `@SpringBootApplication`) built with Maven (`pom.xml`, `mvnw`). Layered package layout under `com.ceb.billing`: `controllers/` (REST endpoints per domain: Auth, AdminUser, Customer, Billing, Approval, Report, Prediction, StagingChange, etc.), `services/` (business orchestration — notably the large `MultiFileImportService` driving a 3-step Master Data → CEB Assist → NGEN pipeline), `repositories/` (Spring Data JPA interfaces mirroring entities), `entities/` (JPA `@Entity` models persisted to MySQL via `spring.jpa.hibernate.ddl-auto=update`), `models/` (DTOs like `LoginRequest`, `JwtResponse`, `ExcelValidationError`), `config/` (Spring Security + JWT filter chain in `WebSecurityConfig`, `AuthTokenFilter`, `AuthEntryPointJwt`, `AuthAccessDeniedHandler`, `UserDetailsServiceImpl`, `JwtUtils`), and `utils/`. Security is stateless: BCrypt passwords, JWT signed with a hex secret from `application.properties`, role-based URL guards (`/api/admin/**` ADMIN, `/api/officer/**` OFFICER|ADMIN, `/api/user/**` USER|ADMIN). Apache POI parses `.xls/.xlsx`; H2 is on the runtime classpath for zero-config demo runs alongside the configured MySQL driver. Seed data is bootstrapped by `DatabaseSeeder` / `ImportTemplateSeedService` and an ad-hoc `AuthController.setup` endpoint that creates default users when none exist.
