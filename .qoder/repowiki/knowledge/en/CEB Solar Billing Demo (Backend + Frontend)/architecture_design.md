Two sibling services share one data contract defined in `schema.sql` (users, customers, upload_history, billing_records, approval_requests, audit_logs); the backend (`backend/`) persists it via Spring Data JPA against MySQL/H2, while the frontend (`frontend/`) consumes its JWT-secured REST endpoints. `generate_test_excel.py` produces a sample multi-sheet workbook that exercises the same column layout the backend's Excel importer expects, and `schema.sql` also seeds default admin/officer/viewer accounts used by both sides' role-based UIs. There is no cross-process orchestration script at this level — each child is started independently.
