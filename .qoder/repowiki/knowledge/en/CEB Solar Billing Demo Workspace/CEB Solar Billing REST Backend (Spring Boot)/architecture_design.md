Flat Maven module (`pom.xml`, `mvnw`) with a single `@SpringBootApplication` entry point in `BillingApplication`. Package layout under `com.ceb.billing` follows the classic layered pattern: `controllers/` (thin `@RestController`s per domain — Auth, AdminUser, Customer, Billing, Approval, Report, Prediction, StagingChange, etc.) delegate to `services/`; `services/` orchestrate business logic and call `repositories/` (Spring Data JPA interfaces) which persist `entities/` (`@Entity` classes like `Customer`, `BillingRecord`, `ImportSession`, `AuditLog`). `models/` holds DTOs (`LoginRequest`, `JwtResponse`, `ExcelValidationError`) shared between controllers and services. Security is configured centrally in `config/WebSecurityConfig` using `@EnableMethodSecurity` with role-based URL guards (`/api/admin/**` ADMIN, `/api/officer/**` OFFICER|ADMIN, `/api/user/**` USER|ADMIN), stateless sessions, BCrypt password encoding, and a custom `AuthTokenFilter` inserted before `UsernamePasswordAuthenticationFilter`; `AuthEntryPointJwt` and `AuthAccessDeniedHandler` return JSON error responses. Database access uses `spring.jpa.hibernate.ddl-auto=update` against MySQL (`application.properties`), with H2 on the runtime classpath for zero-config demos. Apache POI (`poi` + `poi-ooxml` 5.2.5) parses `.xls/.xlsx` files; the heavy `MultiFileImportService` drives a documented 3-step pipeline (Master Data → CEB Assist → NGEN) reusing `PreviewService`'s auto-detection helpers (`findHeaderRowIndex`, `autoDetectColumns`). Seed data is bootstrapped by `DatabaseSeeder` / `ImportTemplateSeedService` plus an ad-hoc `AuthController.setup` endpoint that creates default users when none exist.
